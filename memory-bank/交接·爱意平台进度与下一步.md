## 踩过的坑（别重蹈！）
- **中文乱码**：流式要"增量 UTF-8 解码"(`chat_ai.py` 用 `codecs.getincrementaldecoder`)，且 app.py 的 SSE **yield 字节**(`.encode("utf-8")`)。已修，别改回。
- **Service Worker 必须根作用域**：经 `/sw.js`（app.py 有路由）提供并 `register("/sw.js")`，否则 `serviceWorker.ready` 卡死。
- **iOS 推送**：必须先「添加到主屏幕」、从图标打开的 app 里操作；`Notification.requestPermission()` 要在点击的**同步上下文第一时间**调用(别放在 await 之后)；需 iOS 16.4+。
- **端口**：国内直连国外 8000 端口会被拦 → 用 nginx 反代 80/443 + 域名 HTTPS（已配）。gunicorn 绑 127.0.0.1:8000。
- **Flask session**：多 worker 必须设固定 `SECRET_KEY`(在 .env)，否则老让人重输口令。
- **改前端后**：iPhone 要"删图标→Safari 重新添加到主屏幕"才拿到新代码(缓存狠)。
