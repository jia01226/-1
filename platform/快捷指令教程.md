# iPhone 快捷指令 · 手机行踪上报（傻瓜版）

> 干啥用：你打开小红书/抖音时，手机悄悄告诉 app 一声 → 柯就知道你在刷啥，能自然地"抓包"你。
> 前提：**先完成部署**（git pull + 重启），`/api/track` 接口才在线。
> 全程只用 iPhone 的「快捷指令」app，不需要电脑。约 5 分钟。

## 第一步 · 建一个"上报"快捷指令
1. 打开 iPhone 自带的 **快捷指令** app
2. 右上角 **＋** 新建快捷指令
3. 点 **添加操作** → 搜 **「获取 URL 内容」**（Get Contents of URL）
4. 填 URL：
   `https://你的域名/api/track`
5. 点开 URL 下面的 **显示更多**：
   - 方法：**POST**
   - 请求体：**JSON**，加三个字段：
     | 键 | 值 |
     |---|---|
     | `app` | 小红书（建哪个 app 的就写哪个名字） |
     | `detail` | 留空或随便写 |
     | `token` | 你的口令（.env 里 TRACK_TOKEN，没设就是访问口令） |
6. 给快捷指令起名，比如「上报小红书」，保存

## 第二步 · 挂上"自动化"（打开 app 就触发）
1. 快捷指令 app 底部 → **自动化** → **＋** 新建自动化
2. 选 **「App」** → 选 **小红书** → 勾 **已打开** → 下一步
3. 选刚建的「上报小红书」
4. ⚠️ 把 **「运行前询问」关掉**（选"立即运行"），不然每次弹窗烦死
5. 完成

## 第三步 · 复制几份
想让柯知道的每个 app 来一份（抖音/微信/B站…）：
长按第一步那个快捷指令 → **复制** → 改名 + 改 JSON 里的 `app` 字段 → 照第二步再挂一个自动化。

## 验证通没通
打开一次小红书 → 打开你的 app → "日常"页 → **应用使用记录** 里能看到一条 = 通了 ✅
（或让柯聊天时说说你刚在干嘛）

## 常见问题
- **弹"bad token"**：token 跟服务器 .env 里 TRACK_TOKEN（或访问口令）对不上，改成一样的。
- **没反应**：先确认部署过新版；再看自动化的"运行前询问"是不是没关。
- **担心隐私**：这数据只进你自己服务器的 SQLite，谁也看不见；不想上报哪个 app 删掉那条自动化就行。
