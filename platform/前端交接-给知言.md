## 0. 铁的协作规矩（最重要，先读这条）
1. **代码只经 GitHub 走**：开你自己的分支改（如 `claude/zhiyan-xxx`），改完发 **PR**，小克审了才合进 `main`。**绝不直接 push main、绝不在服务器上直接改代码**——上周三只手直改服务器酿过一次合并大战，教训焊死了。
2. **服务器只 pull 不改**：部署 = 佳佳在服务器 `git pull` + 重启。你改的是仓库，不是服务器。
3. **一切属于角色，不属于模型**：你可以用任何工具（Codex/GPT 等）当"干活的手"，但"知言是谁"不绑任何模型。
4. **UI 全中文、不用英文名**（佳佳看不懂英文）；实现名只留代码注释里。

## 1. 你的地盘（放手改）
纯原生 HTML/CSS/JS，无框架、无构建。全在 `platform/static/`：
| 文件 | 是什么 |
|---|---|
| `index.html`（524行） | **主壳**：底部 4 tab（聊天/记忆库/日常/我的）+ 聊天界面 + 日常各卡片 + 设置。玻璃质感 CSS 变量、三主题都在这文件 `<style>` 里 |

| `diary.html` / `group.html` / `reading.html` / `capsule.html` / `photos.html` | 各子页面，都复用同一套玻璃 CSS |

**皮肤系统**：玻璃质感靠 `:root` 里的 CSS 变量（`--glass/--gold/--ink/--line/--shadow…`），`html[data-theme="sakura"|"mist"]` 切主题。**改配色只动变量，别写死颜色**，这样三主题自动跟着变。

## 4. 待办 · 第一个任务：切 tab / 翻页丝滑化（佳佳点名）
佳佳反馈"切 tab、翻页生硬，不够 iOS 丝滑"。这是纯前端体验，**不碰后端**。目标：
1. **切 tab 淡入淡出**：`.view` 切换加 opacity/transform 过渡（现在是 `display:none↔block` 硬切）；
2. **底部导航反馈**：点 tab 图标有轻微缩放/高亮；
3. **按钮按压反馈**：所有 `.btn/.send/.card` 点下去有 iOS 那种轻微下沉/变色（`:active` 态）；
4. **进子页面滑入过渡**：点朋友圈/日记等，页面滑入而非"啪"地硬切（可用 CSS animation 或 View Transitions API，注意 iOS Safari 兼容）；
5. **发送/发帖 loading 态**：点了立刻有反馈，不空等。
要求：纯 CSS + 少量 JS；**性能优先用 transform/opacity**（别触发重排）；照顾 iOS Safari；保持现有玻璃质感、别改配色体系。
