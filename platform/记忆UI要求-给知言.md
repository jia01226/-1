## 一、要做的 5 块（柯定的顺序）

### 1. 待确认收件箱（最优先）
- **是什么**：机器（柯/系统）猜出来的记忆先进这里，**佳佳点头才算数**，不直接当"佳佳就是这样"。
- **列表项**：一句话内容 ＋ 来源（"柯记的"/"系统总结的"）＋ 重要度 ＋ 时间。
- **动作**：`确认保存` / `改一下再存` / `不保存`（三选一，最好左右滑）。
- **入口红点**：有待确认时，记忆 Tab 上挂个数字角标（催一下佳佳）。

### 2. 记忆卡片库（截图那页）
- **顶部 Tab**：对应我们的分类（别用 Ombre/Notes 英文，换人话：如 `全部/喜好/事件/约定/纪念日`）。
- **搜索框**：搜记忆（防抖，PDF 里那套搜索手法可借）。
- **筛选胶囊**：`全部 / 常驻 / 归档 / 已忘掉(冷静期)`（对应状态，见三·状态）。
- **卡片**：内容摘要 ＋ 标签 ＋ 时间 ＋ **重要度圆点**（1~5 颗）＋ **状态徽章**（见三）。截图那个视觉直接用，很好看。

### 3. 卡片详情与纠错
- 点开一张卡：完整内容 ＋ **"为什么记住"（来源）** ＋ **"何时被用过"（激活记录）**。
- **纠错动作**（这块是纠错闭环的命根子）：`记错了(改内容)` / `改可见范围` / `归档` / `忘掉(进七天冷静期)`。
- **改可见范围** 是个选择器：`只有单聊能提 / 群聊也能提 / 谁都别提(仅后台)`——对应后端 scope，**这是隐私的最后一道人肉闸**。

### 4. 聊天里长按 →"柯想起了什么"
- 柯回话时，长按某条，能看**这轮他调用了哪几条记忆**（透明、可纠错：发现他提了不该提的，一键跳到那张卡改可见范围）。

### 5. 记忆星空（远期，二期再说）
- 可视化，先不急，占个位。

## 二、数据契约（一张卡长这样，程序字段→人话）

| 后端字段 | 人话（UI 显示） | 取值 |
|---|---|---|
| content | 记忆内容 | 一句话 |
| topic | 主题 | 如"家装""健康" |
| importance | 重要度圆点 | 1~5 |
| source | 为什么记住 | 佳佳亲口说 / 柯记的 / 系统总结 / 老记忆 |
| status | 状态徽章 | 常驻 / 已被更新 / 已归档 / 待确认 / 忘掉冷静期 |
| scope | 可见范围（纠错用） | 只单聊 / 群聊也可 / 谁都别提 |
| review_state | — | 待佳佳确认 / 已确认（收件箱用） |
| created_at | 时间 | — |

- **状态徽章配色建议**：常驻=樱粉、待确认=琥珀、忘掉冷静期=灰蓝、归档=浅灰。截图里 `resolved` 那种小灰标就是范式。

## 三、我要向你提的 UI 要求（后端视角）

1. **可见范围选择器必须显眼**：这是佳佳唯一能人肉兜底隐私的地方，别藏太深。
2. **"忘掉"要有七天可找回的提示**：点忘掉后给一句"7 天内随时找回"，别让她以为一点就永久没了（柯特意为她低谷时冲动加的保险，UI 要把这份安全感传达到）。
3. **待确认红点**：有 pending 就催，别让机器猜的记忆默默堆着。
4. **Breath Lab / 记忆诊断页** 挂在 `设置 → 开发者模式` 里，日常界面别露技术信息（token/得分那些）。佳佳日常看不到，我调试时才进。
5. **别在前端做隐私筛选**：哪条能在群聊出现，后端查询层已经焊死了，你**照拿照显**就行，不用也不要在前端二次过滤（重复逻辑＝将来对不上）。

## 四、接口（形态先定，HTTP 我下一批补，你可先 mock）

```
GET  /api/memory/pending          待确认列表
POST /api/memory/pending/review   {id, store, approve, content?}  确认/改/不保存
GET  /api/memory/cards            卡片库（?tab= &q= &filter=）
GET  /api/memory/card/<id>        详情（含来源+激活记录）
POST /api/memory/card/<id>/scope  {scope}  改可见范围
POST /api/memory/card/<id>/forget 忘掉（进冷静期）
POST /api/memory/card/<id>/recover 找回
GET  /api/memory/breath?q=        诊断页数据（开发者模式）
```
