## 一、这轮目标

- 产品继续以“关系优先”为核心，不做 AI 管理后台。
- 视觉改为暖奶白、浅暖灰、低饱和酒红、少量香槟金。
- 底部一级导航收敛为：`柯 / 回忆 / 我们的`。
- 设置、用量、推送、图片管理从一级导航移到右上角“佳”头像入口。
- 手机优先：点击区域至少 44px，底部导航固定，详情和消息菜单使用底部抽屉，适配安全区。

## 二、前端变化

### `static/index.html`

- 移除首页口令输入门；若服务器仍配置 `ACCESS_PASSCODE`，前端只提示去部署设置关闭，不再收集口令。
- 日记、共读、时间胶囊、图片管理、待确认、珍藏页和朋友圈的历史口令输入门也一并移除。
- 聊天请求新增 `model` 字段，值来自模型选择器；仍由后端白名单兜底。
- 长按柯的消息改为：`柯想起了什么 / 复制 / 重新回复 / 继续说下去`。
- “柯想起了什么”通过底部半屏抽屉展示自然语言回忆，不显示相似度、向量分数等字段。
- 直接加载 `static/ui-redesign.js`，不再只依赖 service worker 注入。
- 修复大/特大字号使用整页缩放后把底部导航挤出屏幕的问题：缩放时同步补偿页面宽高，内容区允许收缩滚动，底栏始终留在视口内。

### `static/ui-redesign.js`

- 重建统一暖白页面骨架、三栏导航、头像设置入口。
- 聊天输入区调用 `GET /api/models`，展示真实模型白名单并记住上次选择。
- 回忆中心四页：待确认、卡片库、时间线、回忆星图。
- 待确认数字角标只在有待确认项目时显示。
- 卡片库调用现有 `GET /api/memory/cards`；详情调用 `GET /api/memory/card`。
- 详情纠错继续使用现有 `/edit`、`/forget` 接口，界面文案为人话。
- 星图使用深酒红底、暖白/胭粉/香槟金节点；中心为“佳佳和柯”，含主题点击、缩放和时间滑杆。

### `static/inbox.html`

### `static/treasure.html`

- 移除该页口令输入门。

## 三、本轮唯一后端接口变化

新增：

```http
POST /api/moments/comment/edit
Content-Type: application/json

{"id": 12, "content": "修改后的评论"}
```

返回：

```json
{"ok": true}
```

约束：

- 空内容返回 400。
- 不存在或不是 `author=user` 的评论返回 404。
- 界面不能修改柯的评论，避免把柯的发言改写成佳佳的意思。
- 数据层函数：`db.edit_comment(cid, content, author="user")`。

## 四、模型切换约定

- 前端只读取 `GET /api/models` 返回的 `models` 与 `default`。
- 选择结果随 `/api/chat` 的 `model` 字段发送。
- `chat_ai.resolve_model()` 继续负责白名单校验，前端不绕过。
- 服务器要出现多个选项，需要在 `.env` 配置 `MODEL_WHITELIST`；没有配置时仍只显示默认模型。

1. `/api/moments/comment/edit` 的作者限制是否符合现有权限边界。
2. 生产环境是否已清空 `ACCESS_PASSCODE`；前端已不再提供口令输入框。
3. `/api/memory/injection-log` 返回形态是否能稳定按 `msg_id` 对上聊天消息。当前 UI 做了兼容和友好兜底，但若能补一个面向普通界面的“某条回复用了哪些记忆”接口会更稳。
4. 部署后更新 service worker 缓存版本，避免旧 `ui-redesign.js` 滞留。
